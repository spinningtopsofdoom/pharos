Summary

Associates key(s) to value(s) in a collection

Details

If the same key appears multiple times, the last key value pair is the one that gets associated in the collection. (i.e. (assoc {:a :b} :a :c :a :d) => {:a :d}) If the value is a map then a new map of the same type is returned that contains the mapping for key(s) to value(s). If the value is a vector then only integer keys up to the number of items in a vector are allowed.

Examples

(assoc {:key1 :val1, :key2 :val2} :key1 :foo :key1 :bar :key3 :bla) (assoc nil :key1 :foo :key1 :bar :key3 :bla) (assoc {} :key1 :foo :key1 :bar :key3 :bla) (assoc [1 2 3] 0 42 2 12) (assoc [1 2 3] 0 42 0 88) (assoc [1 2 3] 3 42)

Failures

Any other keys will throw an exception. (e.g. [42 314 2878] only allows keys 0 1 2 3 and [42] only allows keys 0 and 1. (assoc [42] 65 22) or (assoc [42] :foo 22) will throw an exception)

Failure Examples

(assoc [1 2 3] 8 42)
